# BACKLOG & ARCHITECTURAL IMPROVEMENTS


## Match History Optimization
*   **Goal:** Improve performance and file I/O as match history grows.
*   **Proposed Architecture:** Move from a single monolith JSON per PUUID (`data/matches/{puuid}.json`) to a "Global Match Store".
    *   `data/matches/details/{match_id}.json`: Individual files per match to prevent large file rewrites.
    *   `data/matches/indices/{puuid}.json`: A lightweight index mapping PUUIDs to Match IDs + Timestamps + Queue IDs.
*   **Benefits:** Atomic saves, millisecond decay calculation, and shared match deduplication.

## Asset & Static Data Caching
*   [ ] **Local Asset Manager:** Automatically download and cache champion/item icons locally.
*   [ ] **Version-Gate Sync:** Only refresh static data (`champion.json`, `queues.json`) when a patch version increment is detected.
*   [ ] **ID-to-Name Mapping:** Build a local lookup table for Maps, Queues, and Champions to replace hardcoded dictionaries.
